The Hook Engine — Industrial React utilities

Focused, composable hooks that operate like the invisible systems powering modern web apps. Lightweight, testable, and engineered for precision.

$ npm install use-web-kit

About

use-web-kit bundles a set of focused React hooks that solve routine web problems: detecting connectivity changes, queuing idle work, observing intersections, broadcasting small state across tabs, handling page lifecycle events, and adaptive polling. Each hook aims to be minimal, well-documented, and easy to compose.

They’re useful to reduce boilerplate in apps and keep your components declarative and testable.

Installation

Install with your package manager of choice.

npm install use-web-kit
yarn add use-web-kit
pnpm add use-web-kit

Signature

function useAdaptivePolling(
    fn: () => Promise<T> | T,
    options?: {
        interval?: number; // base ms
        maxInterval?: number;
        activeInterval?: number;
        pauseOnHidden?: boolean;
    }
): { value?: T; start: () => void; stop: () => void }

Parameters

Name Type Default Description
fn () => Promise<T> | T — Function returning data to poll.
options.interval number 5000 Base polling interval in ms.
options.maxInterval number 60000 Maximum backoff interval.
options.activeInterval number 1000 Faster interval while active (if used).
options.pauseOnHidden boolean true If true, pauses polling when document is hidden.

Returns

Name Type Description
value T | undefined Latest value returned by fn.
start () => void Start polling manually.
stop () => void Stop polling.

Example (Basic)

// AdaptivePollingExample.jsx
import React from 'react'
import { useAdaptivePolling } from 'use-web-kit'

export default function AdaptivePollingExample(){
    const { value } = useAdaptivePolling(async ()=>{
        const r = await fetch('https://jsonplaceholder.typicode.com/todos/1')
        const json = await r.json()
        return json.title
    }, { interval: 5000, maxInterval: 60000 })

    return (
        

            Server status:
            
{String(value ?? 'loading...')}

        

    )
}

Behavior Explanation

On mount the hook starts a timer calling fn at interval ms. If the page is idle or hidden and pauseOnHidden is true, polling pauses. When responses are successful, the hook updates value. When failures occur the hook can increase the interval up to maxInterval to reduce load.

Example (Advanced)

// Adaptive with manual control
import React from 'react'
import { useAdaptivePolling } from 'use-web-kit'

export default function AdaptiveAdvanced(){
    const { value, start, stop } = useAdaptivePolling(()=> fetch('/api/health').then(r=>r.json()), { interval:10000, pauseOnHidden:true })
    return (
        

            
Health: {String(value ?? 'unknown')}

            

                
                
            

        

    )
}

Edge Cases

  • Network flapping may cause frequent retries; consider exponential backoff in fn.
  • Browsers may throttle timers in background tabs; rely on pauseOnHidden where appropriate.

Best Practices

  • Prefer WebSocket/SSE for real-time data when possible.
  • Use pauseOnHidden to reduce battery and network usage.

useBroadcastState

Share small serializable state across tabs/windows via BroadcastChannel with a localStorage fallback. Useful for theme, active-tab flags, or lightweight cross-tab coordination.

Signature

function useBroadcastState(key: string, initial: T, options?: { serialize?: (v: T) => string; deserialize?: (s: string) => T }): [T, (v: T) => void]

Parameters

Name Type Default Description
key string — Storage key / channel name.
initial T — Initial value when no state exists.
options object {} Serializer/deserializer overrides.

Returns

Name Type Description
state T Current shared state.
setState (v: T) => void Update and broadcast the state.

Example (Basic)

// BroadcastExample.jsx
import React from 'react'
import { useBroadcastState } from 'use-web-kit'

export default function BroadcastExample(){
    const [theme, setTheme] = useBroadcastState('site:theme', 'light')
    return (
        

            
Theme: {theme}

            
        

    )
}

Behavior Explanation

The hook writes serialized state to a BroadcastChannel (or localStorage) and listens for updates. When another tab broadcasts the same key, the hook updates local state immediately.

Example (Advanced)

// Broadcast with custom serializer
import React from 'react'
import { useBroadcastState } from 'use-web-kit'

export default function BroadcastAdvanced(){
    const [value, setValue] = useBroadcastState('app:count', 0, { serialize: v => String(v), deserialize: s => Number(s) })
    return (
        

            
Count: {value}

            
        

    )
}

Edge Cases

  • Large payloads cause slow serialization and may block the main thread—keep messages small.
  • Older browsers may not support BroadcastChannel; localStorage fallback introduces storage event semantics.

Best Practices

  • Keep broadcast values small and serializable (strings, numbers, small objects).
  • Use for UI flags and lightweight coordination rather than large data sync.

useIdleQueue

Schedule low-priority work to run when the browser is idle. Falls back to setTimeout if requestIdleCallback is unavailable.

Signature

function useIdleQueue(): (task: () => void, options?: { timeout?: number }) => void

Parameters

Name Type Default Description
task () => void — Function to run during idle time.
options.timeout number undefined Maximum ms to wait before running the task.

Returns

Name Type Description
run (task: () => void, options?: { timeout?: number }) => void Schedules task to run during idle time (or after timeout).

Example (Basic)

// IdleExample.jsx
import React from 'react'
import { useIdleQueue } from 'use-web-kit'

export default function IdleExample(){
    const run = useIdleQueue()
    return (
        

            
        

    )
}

Behavior Explanation

The hook exposes a run function that schedules tasks using requestIdleCallback. If unavailable, it falls back to setTimeout. A timeout option ensures critical work runs even if the browser never reports idle time.

Example (Advanced)

// Idle with timeout fallback
import React from 'react'
import { useIdleQueue } from 'use-web-kit'

export default function IdleAdvanced(){
    const run = useIdleQueue()
    function heavy(){
        // do non-blocking chunked work
        console.log('running heavy task')
    }
    return (
        

            
        

    )
}

Edge Cases

  • On very busy pages idle callbacks may be deferred indefinitely; use timeout for critical tasks.

Best Practices

  • Use idle tasks for analytics aggregation, cache cleanup, or prefetching non-critical assets.

Returns

Name Type Description
ref (el: Element | null) => void Attach to observed element.

useNetworkStatus

Track online/offline status. Use to gate network requests, show sync state, or trigger retries when connectivity returns.

Signature

function useNetworkStatus(): boolean

Parameters

This hook has no parameters.

Returns

Name Type Description
online boolean True when the browser reports network connectivity.

Example (Basic)

// NetworkExample.jsx
import React from 'react'
import { useNetworkStatus } from 'use-web-kit'

export default function NetworkExample(){
    const online = useNetworkStatus()
    return (
        

            Online:
            
{String(online)}

        

    )
}

Behavior Explanation

The hook listens to the browser online and offline events and returns a boolean reflecting the current state. Note that this reflects the browser's connectivity status and not guaranteed end-to-end reachability.

Example (Advanced)

// Queue requests while offline and flush when online
import React, {useState} from 'react'
import { useNetworkStatus } from 'use-web-kit'

export default function NetworkAdvanced(){
    const online = useNetworkStatus()
    const [queue, setQueue] = useState([])

    async function send(data){
        if(!online){
            setQueue(prev => [...prev, data])
            return
        }
        await fetch('/api/submit', { method:'POST', body: JSON.stringify(data) })
    }

    // simple flush when online
    React.useEffect(()=>{
        if(online && queue.length){
            queue.forEach(item => fetch('/api/submit', { method:'POST', body: JSON.stringify(item) }))
            setQueue([])
        }
    }, [online])

    return (
        

            
Status: {String(online)}

            
        

    )
}

Edge Cases

  • The browser's online status may be true even if the server is unreachable (captive portals, DNS issues).

Best Practices

  • Use this hook for UI decisions and light retry logic; perform an application-level health check for true reachability.

usePageLifecycle

Expose document visibility and focus lifecycle states. Use to pause timers, mute updates, or defer work when the page is not active.

Signature

function usePageLifecycle(): DocumentVisibilityState

Parameters

This hook has no parameters.

Returns

Name Type Description
state DocumentVisibilityState One of 'visible', 'hidden', 'prerender', etc., matching document.visibilityState.

Example (Basic)

// PageLifecycleExample.jsx
import React from 'react'
import { usePageLifecycle } from 'use-web-kit'

export default function PageLifecycleExample(){
    const state = usePageLifecycle()
    return (
        

            
Visibility: {state}

        

    )
}

Behavior Explanation

The hook subscribes to visibility and focus events and returns the current document.visibilityState. Use it to pause animations, stop polling, or reduce background work.

Example (Advanced)

// Pause an interval when not visible
import React, {useEffect, useState} from 'react'
import { usePageLifecycle } from 'use-web-kit'

export default function LifecycleAdvanced(){
    const state = usePageLifecycle()
    const [count, setCount] = useState(0)

    useEffect(()=>{
        if(state !== 'visible') return
        const id = setInterval(()=> setCount(c=>c+1), 1000)
        return () => clearInterval(id)
    }, [state])

    return (
        

            
Counter (runs only when visible): {count}

        

    )
}

Edge Cases

  • Some browsers may report different lifecycle states while devtools are open; test on target platforms.

Best Practices

  • Use lifecycle hooks to reduce CPU and network usage when your app is not visible.
