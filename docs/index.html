About use-web-kit

A set of focused React hooks that solve routine web problems. Minimal, well-documented, and easy to compose — designed to keep your components declarative and testable.

Lightweight

Zero external dependencies. Ships under 3 kB gzipped. Tree-shakeable so you only pay for what you import.

TypeScript Native

Written in TypeScript with full type definitions. IntelliSense works out of the box — no @types package needed.

Composable

Each hook solves one problem well. Combine them freely — use network status to gate polling, or idle queues to defer analytics.

Well Tested

Full Jest test coverage for every hook. Designed for deterministic behavior with proper cleanup on unmount.

Quick Start

Minimal working example using useNetworkStatus. Copy-paste into a React app.

// QuickStart.jsx
import React from 'react'
import { useNetworkStatus } from 'use-web-kit'

export default function QuickStart() {
  const online = useNetworkStatus()
  return (
    <main style={{ padding: 20 }}>
      <h3>Network</h3>
      <p>Status: {online ? 'Online' : 'Offline'}</p>
    </main>
  )
}

What happens: the hook subscribes to the browser online/offline events and returns a boolean. Use it to toggle UI or queue sync tasks when connectivity returns.

useAdaptivePolling

Smart polling that adapts to page visibility and error state. Backs off on failure, pauses when the document is hidden, and provides manual start/stop controls.

Signature

function useAdaptivePolling<T = any>(
    fn: () => Promise<T> | T,
    options?: {
        interval?: number;       // base ms
        maxInterval?: number;
        activeInterval?: number;
        pauseOnHidden?: boolean;
    }
): { value?: T; start: () => void; stop: () => void }

Parameters

NAME TYPE DEFAULT DESCRIPTION
fn () => Promise<T> | T — Function returning data to poll.
options.interval number 5000 Base polling interval in ms.
options.maxInterval number 60000 Maximum backoff interval.
options.activeInterval number 1000 Faster interval while active.
options.pauseOnHidden boolean true Pause polling when document is hidden.

Returns

NAME TYPE DESCRIPTION
value T | undefined Latest value returned by fn.
start () => void Start polling manually.
stop () => void Stop polling.

Example — Basic

// AdaptivePollingExample.jsx
import React from 'react'
import { useAdaptivePolling } from 'use-web-kit'

export default function AdaptivePollingExample() {
    const { value } = useAdaptivePolling(async () => {
        const r = await fetch('https://jsonplaceholder.typicode.com/todos/1')
        return (await r.json()).title
    }, { interval: 5000, maxInterval: 60000 })

    return (
        <div>
            <strong>Server status:</strong>
            <div>{String(value ?? 'loading...')}</div>
        </div>
    )
}

Behavior

On mount the hook starts a timer calling fn at interval ms. If the page is idle or hidden and pauseOnHidden is true, polling pauses. When responses are successful, the hook updates value. On failure the interval increases up to maxInterval to reduce load.

Example — Advanced (Manual Control)

import React from 'react'
import { useAdaptivePolling } from 'use-web-kit'

export default function AdaptiveAdvanced() {
    const { value, start, stop } = useAdaptivePolling(
        () => fetch('/api/health').then(r => r.json()),
        { interval: 10000, pauseOnHidden: true }
    )
    return (
        <div>
            <div>Health: {String(value ?? 'unknown')}</div>
            <button onClick={start}>Start</button>
            <button onClick={stop}>Stop</button>
        </div>
    )
}

Edge Cases

  • Network flapping may cause frequent retries; consider exponential backoff in fn.
  • Browsers may throttle timers in background tabs; rely on pauseOnHidden.

Best Practices

  • Prefer WebSocket/SSE for real-time data when possible.
  • Use pauseOnHidden to reduce battery and network usage.
SSR Safety: This hook accesses document.visibilityState and setInterval. Guard with typeof window !== 'undefined' or call only inside useEffect. In Next.js / Remix, the hook is safe as-is because React hooks run client-side only.

useBroadcastState

Share small serializable state across tabs/windows via BroadcastChannel with a localStorage fallback. Useful for theme, active-tab flags, or lightweight cross-tab coordination.

Signature

function useBroadcastState<T>(
    key: string,
    initial: T,
    options?: {
        serialize?: (v: T) => string;
        deserialize?: (s: string) => T
    }
): [T, (v: T) => void]

Parameters

NAME TYPE DEFAULT DESCRIPTION
key string — Storage key / channel name.
initial T — Initial value when no state exists.
options object {} Serializer / deserializer overrides.

Returns

NAME TYPE DESCRIPTION
state T Current shared state.
setState (v: T) => void Update and broadcast the state.

Example — Basic

// BroadcastExample.jsx
import React from 'react'
import { useBroadcastState } from 'use-web-kit'

export default function BroadcastExample() {
    const [theme, setTheme] = useBroadcastState('site:theme', 'light')
    return (
        <div>
            <p>Theme: {theme}</p>
            <button onClick={() => setTheme(theme === 'light' ? 'dark' : 'light')}>Toggle</button>
        </div>
    )
}

Behavior

The hook writes serialized state to a BroadcastChannel (or localStorage) and listens for updates. When another tab broadcasts the same key, the hook updates local state immediately.

Example — Advanced (Custom Serializer)

import React from 'react'
import { useBroadcastState } from 'use-web-kit'

export default function BroadcastAdvanced() {
    const [value, setValue] = useBroadcastState('app:count', 0, {
        serialize: v => String(v),
        deserialize: s => Number(s)
    })
    return (
        <div>
            <div>Count: {value}</div>
            <button onClick={() => setValue(value + 1)}>Inc</button>
        </div>
    )
}

Edge Cases

  • Large payloads cause slow serialization and may block the main thread — keep messages small.
  • Older browsers may not support BroadcastChannel; localStorage fallback uses storage events.

Best Practices

  • Keep broadcast values small and serializable (strings, numbers, small objects).
  • Use for UI flags and lightweight coordination rather than large data sync.
SSR Safety: This hook uses BroadcastChannel and localStorage, which are browser-only APIs. Guard with typeof window !== 'undefined' or rely on React’s client-side hook execution. Falls back gracefully in environments without BroadcastChannel.

useIdleQueue

Schedule low-priority work to run when the browser is idle. Falls back to setTimeout if requestIdleCallback is unavailable.

Signature

function useIdleQueue(): (task: () => void, options?: { timeout?: number }) => void

Parameters

NAME TYPE DEFAULT DESCRIPTION
task () => void — Function to run during idle time.
options.timeout number undefined Max ms to wait before running.

Returns

NAME TYPE DESCRIPTION
run (task, options?) => void Schedules task to run during idle time (or after timeout).

Example — Basic

// IdleExample.jsx
import React from 'react'
import { useIdleQueue } from 'use-web-kit'

export default function IdleExample() {
    const run = useIdleQueue()
    return (
        <button onClick={() => run(() => console.log('idle task ran'))}>
            Schedule
        </button>
    )
}

Behavior

The hook exposes a run function that schedules tasks using requestIdleCallback. If unavailable, it falls back to setTimeout. A timeout option ensures critical work runs even if the browser never reports idle time.

Example — Advanced (With Timeout)

import React from 'react'
import { useIdleQueue } from 'use-web-kit'

export default function IdleAdvanced() {
    const run = useIdleQueue()
    return (
        <button onClick={() => run(() => console.log('heavy task'), { timeout: 2000 })}>
            Run with timeout
        </button>
    )
}

Edge Cases

  • On very busy pages idle callbacks may be deferred indefinitely; use timeout for critical tasks.

Best Practices

  • Use idle tasks for analytics aggregation, cache cleanup, or prefetching non-critical assets.
SSR Safety: This hook uses requestIdleCallback (with a setTimeout fallback). Both APIs are browser-only. Guard with typeof window !== 'undefined' or rely on React’s client-side hook execution in Next.js / Remix.

useIntersection

Observe element visibility in the viewport. Useful for lazy loading, analytics, or triggering work only when an element becomes visible.

Signature

function useIntersection(
    options?: IntersectionObserverInit
): { ref: (el: Element | null) => void; isIntersecting: boolean }

Parameters

NAME TYPE DEFAULT DESCRIPTION
options IntersectionObserverInit {} Observer options: root, rootMargin, threshold.

Returns

NAME TYPE DESCRIPTION
ref (el: Element | null) => void Attach to observed element.
isIntersecting boolean True when element intersects the viewport.

Example — Basic

// IntersectionExample.jsx
import React from 'react'
import { useIntersection } from 'use-web-kit'

export default function IntersectionExample() {
    const { ref, isIntersecting } = useIntersection({ threshold: 0.5 })
    return (
        <div ref={ref} style={{ height: 200 }}>
            Visible: {String(isIntersecting)}
        </div>
    )
}

Behavior

The hook creates an IntersectionObserver and updates isIntersecting when the element meets the provided threshold. The ref setter detaches any previous element and observes the new one.

Example — Advanced (Lazy Image)

// LazyImage.jsx
import React, { useState } from 'react'
import { useIntersection } from 'use-web-kit'

export default function LazyImage({ src, alt }) {
    const { ref, isIntersecting } = useIntersection({ threshold: 0.1 })
    const [loaded, setLoaded] = useState(false)
    return (
        <div ref={ref} style={{ minHeight: 200 }}>
            {isIntersecting && (
                <img src={src} alt={alt}
                     onLoad={() => setLoaded(true)}
                     style={{ opacity: loaded ? 1 : 0.6 }} />
            )}
        </div>
    )
}

Edge Cases

  • IntersectionObserver is not available in very old browsers; include a polyfill if needed.

Best Practices

  • Prefer threshold arrays for progressive reveals; keep observers scoped to necessary elements.
SSR Safety: This hook creates an IntersectionObserver, a browser-only API. The hook initializes the observer inside useEffect, so it is safe for SSR frameworks like Next.js and Remix out of the box.

useNetworkStatus

Track online/offline status. Use to gate network requests, show sync state, or trigger retries when connectivity returns.

Signature

function useNetworkStatus(): boolean

Parameters

This hook has no parameters.

Returns

NAME TYPE DESCRIPTION
online boolean True when the browser reports network connectivity.

Example — Basic

// NetworkExample.jsx
import React from 'react'
import { useNetworkStatus } from 'use-web-kit'

export default function NetworkExample() {
    const online = useNetworkStatus()
    return (
        <div>
            <strong>Online:</strong>
            <div>{String(online)}</div>
        </div>
    )
}

Behavior

The hook listens to the browser online and offline events and returns a boolean reflecting the current state. Note that this reflects the browser's connectivity status and not guaranteed end-to-end reachability.

Example — Advanced (Offline Queue)

// Queue requests while offline and flush when online
import React, { useState, useEffect } from 'react'
import { useNetworkStatus } from 'use-web-kit'

export default function NetworkAdvanced() {
    const online = useNetworkStatus()
    const [queue, setQueue] = useState([])

    useEffect(() => {
        if (online && queue.length) {
            queue.forEach(item =>
                fetch('/api/submit', { method: 'POST', body: JSON.stringify(item) })
            )
            setQueue([])
        }
    }, [online])

    return <div>Status: {String(online)}</div>
}

Edge Cases

  • The browser's online status may be true even if the server is unreachable (captive portals, DNS issues).

Best Practices

  • Use this hook for UI decisions and light retry logic; perform an application-level health check for true reachability.
SSR Safety: This hook reads navigator.onLine and subscribes to window online/offline events. Guard with typeof window !== 'undefined' or rely on React’s client-side hook execution. Defaults to true during SSR.

usePageLifecycle

Expose document visibility and focus lifecycle states. Use to pause timers, mute updates, or defer work when the page is not active.

Signature

function usePageLifecycle(): DocumentVisibilityState

Parameters

This hook has no parameters.

Returns

NAME TYPE DESCRIPTION
state DocumentVisibilityState 'visible', 'hidden', etc., matching document.visibilityState.

Example — Basic

// PageLifecycleExample.jsx
import React from 'react'
import { usePageLifecycle } from 'use-web-kit'

export default function PageLifecycleExample() {
    const state = usePageLifecycle()
    return <div>Visibility: {state}</div>
}

Behavior

The hook subscribes to visibility and focus events and returns the current document.visibilityState. Use it to pause animations, stop polling, or reduce background work.

Example — Advanced (Pause Interval)

import React, { useEffect, useState } from 'react'
import { usePageLifecycle } from 'use-web-kit'

export default function LifecycleAdvanced() {
    const state = usePageLifecycle()
    const [count, setCount] = useState(0)

    useEffect(() => {
        if (state !== 'visible') return
        const id = setInterval(() => setCount(c => c + 1), 1000)
        return () => clearInterval(id)
    }, [state])

    return <div>Counter (visible only): {count}</div>
}

Edge Cases

  • Some browsers may report different lifecycle states while devtools are open.

Best Practices

  • Use lifecycle hooks to reduce CPU and network usage when your app is not visible.
SSR Safety: This hook accesses document.visibilityState and listens to visibilitychange / focus / blur events. Guard with typeof window !== 'undefined' or rely on React’s client-side hook execution. Defaults to "visible" during SSR.
